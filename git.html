command

command move option
init リポジトリの新規作成or初期化
clone リポジトリのクローンを作成
fetch リモートリポジトリの内容を取得
merge 他のブランチやコミットの内容を現在のブランチに取り込む
pull fetch + mergeを行う(注意)
add 変更されたファイルをインデックスに追加する(ステージング)
. 全てのファイル
commit インデックスに追加した変更をリポジトリに記録する(コミット)
(Null) vimが開く
--all(-a) add + commit
-m "x" コメントxをつける
-F- << E0M 複数行に渡ってE0Mと入力するまでコメントを記入できる
--allow-empty 空のコミットを許可する(-m --allow-emptyで空のままコミットできる)
push ローカルリポジトリの変更内容をリモートリポジトリに送信する
mv ファイルやディレクトリの名前を変更する
rm gitで管理しているファイルを削除する
--cached ワークツリーのファイルは保持する
--force(-f) コミット前やインデックスに追加する前のファイルであっても削除する
-r ディレクトリを再帰的に削除する
--dry-run(-n) 実行せずに実行内容を表示する
reset 履歴時間軸を戻す
--soft HEADを戻す(commitの取り消し)
--mixed HEADとインデックスを戻す(commitとaddの取り消し)
--hard 全て(HEADとインデックスとワーキングツリー)を戻す(以前commitした時まで取り消す)
HEAD^(@^) 一つ前のコミット
HEAD~~~,@^^^,HEAD~3,HEAD~{3},HEAD^^^ 3つ前のコミット
ハッシュ値 そこのコミット
status ワークツリーにある現在の状態を表示する
remote リモートリポジトリを関連づける
show ファイルの内容や差分などを表示する
diff コミット同士やコミットとワークツリーの内容を比較する
checkout ワークツリーを異なるブランチに切り替える
branch ブランチを操作する
log コミット時のログを表示する
tag コミットにタグをつけたり
grep リポジトリで管理されているファイルをパターン検索
rebase コミットを最適用する(注意)
config 現在の設定を取得,変更する
reflog resetの取り消し git reset --hard ORIG_HEADで直前のresetの取り消し
bisect 問題の効果的な探索
