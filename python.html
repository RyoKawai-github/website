Anaconda-command

command move option
conda info -e 存在している仮想環境のリストを表示 –envsでも可
conda create -n exp 名前の仮想環境を作成する -nameでも可
conda create -n exp python=3.5 仮想環境expにpython3.5をインストールする. pythonのみだと最新をインストールする. python Anacondaとするとrootにある全てのライブラリが入る
conda activate exp 仮想環境expを起動する windowsはcondaがいらない
conda deactivate 今いる仮想環境を終了する
conda install Lib conda install -c conda-forge Lib pip install Lib Libを今いる仮想環境にインストールする activateをしていない場合はroot環境にはいる. condaのデフォルト場所にあるものは上, conda-forgeにあるものは真ん中, PyPI(Python Package Index)にあるものは下でインストールできる. -cは-channelでも可で, conda-forgeの入っているチャンネルそのライブラリの公式が推奨する方でやるのが良い. どっちでも良い場合はどちらかに統一した方が良いらしい(ごちゃごちゃになると環境が壊れたりするらしい).
conda uninstall Lib conda uninstall -c conda-forge Lib pip uninstall Lib Libを今いる仮想環境からアンインストールする インストールした方でやる. これがわかりやすいという意味でも統一した方が良さそう.
conda list 今いる仮想環境に入っているライブラリの一覧を参照する pip listもあるらしい
conda remove -n exp --all 仮想環境expを削除する

numpy.unique(ndarrayやlist) ndarrayの要素のuniqueなものをソートしてndarray型で出力する.
np.unique(a, return_index=True, return_inverse=True, return_counts=True) ndarrayの要素のuniqueなものをソートしたndarray型(以後u), uの各要素の最初にaに現れたインデックスのndarray型, uの各要素のインデックスをaで表すndarray型, uの各要素の出現回数のndarray型, のタプルが出力される. 多次元配列の場合はフラットで処理されるが, axis=0などとすればその成分ごとにみる. 詳しくはここ u, counts = numpy.unique([0,0,10,10,20,30], return_counts=True) u[counts == 1] #[20,30]
